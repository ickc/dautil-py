dautil.util.df_linregress(df, grouplevel=0, regresslevel=1, regressindex=2, regressorder=2)[source]¶

per level grouplevel, perform a linregress of each column vs. level regresslevel assumes df has MultiIndex of 2 levels, behavior undefined otherwise.

regressindex and regressorder control which amount the 5 outputs from scipy.stats.linregress is returned If regressindex is None, return all as an object, else, only return the stat at this index. In the latter case, the stat will be raised to the power regressorder, which is useful to return $R^2$. dautil.util.get_box(array)[source]¶

array: numpy.ndarray return: per dim. in array, find the min. and max. index s.t. the other dim. are identically zero.

This might be used when one want to trim out the empty “boundary” of an array to reduce size. In principle, a trimmed array with this attribute can reconstruct the original array.

Note:

  • This might not make sense if ndim = 1

  • This is relatively slow, that uses numpy only to walk through the array.

dautil.util.map_parallel(f, *args, **kwargs)[source]¶

equivalent to map(f, *args) processes: no. of parallel processes when multiprocessing is used (in the case of mpi, it is determined by mpiexec/mpirun args) mode:

  • mpi: using mpi4py.futures

  • multiprocessing: using multiprocessing from standard library

  • multithreading: using multithreading from standard library

  • serial: using map

Note: this is for Python 2 compatibility, else map_parallel(f, *args, mode=’multiprocessing’, processes=1)

dautil.util.planck_taper[source]¶

return a planck taper window, mapping from 0..m to 0..1, excluding 0, m index (which equals to 0, and 1 respectively) i.e. the returned array has length m - 1 the reason is that the array is assumed to be zero-padded anyway, where the 0 at the boundary is granted.
